TMC2209 Library And Register Reference

This page gives a compact overview of the most relevant TMC2209 flags and settings exposed by the janelia-arduino/TMC2209 library. The most useful items are sorted to the top. Each parameter cell tells you whether it is a chip field or a library-derived value, where it lives in the chip or the library, and which public library functions are relevant to it. Unless noted otherwise, the defaults below describe the effective library state after a successful setup() on a responding driver, not the raw chip reset defaults.

Settings Table

is_communicating Library value Location: Settings Functions: isCommunicating(), getSettings() True after successful setup with a responding driver IOIN.version Library check that UART communication with the chip is working. If this is False, the driver is not answering properly and almost nothing else can be trusted.
is_setup Library value Location: Settings Functions: isSetupAndCommunicating(), isCommunicatingButNotSetup(), getSettings(), setup() True after successful setup with a responding driver Mainly GCONF.pdn_disable, bit 6 Library-level “setup is good” flag. In this library it mostly means GCONF.pdn_disable is set. Useful quick health check, but it is not a full proof that every register is correct.
pdn_disable Chip flag Location: GCONF / GlobalConfig Functions: setup() True GCONF.pdn_disable, bit 6 Disables PDN behavior and enables proper UART-oriented configuration mode. This is one of the key bits for advanced UART control on the TMC2209.
mstep_reg_select Chip flag Location: GCONF / GlobalConfig Functions: setup(), setMicrostepsPerStep(), setMicrostepsPerStepPowerOfTwo() True GCONF.mstep_reg_select, bit 7 Selects whether microstep resolution comes from hardware pins MS1/MS2 or from the register CHOPCONF.MRES. If this is False, the pins decide microsteps. If this is True, UART/register settings decide microsteps.

Possible values: 1, 2, 4, 8, 16, 32, 64, 128, 256. This directly affects smoothness, resolution, and required step rate from the controller.

If this is False, motor outputs are effectively off even if communication still works.
enable_spread_cycle Chip flag Location: GCONF / GlobalConfig Functions: enableStealthChop(), disableStealthChop() False GCONF.enable_spread_cycle, bit 2 Chooses the current-control mode. Possible values: False = StealthChop, True = SpreadCycle. One of the biggest behavior choices: torque/dynamics versus quiet operation.

irun_register_value Library value Location: Settings Functions: setRunCurrent(), setAllCurrentValues(), getSettings() 0 Cached IHOLD_IRUN.IRUN, bits 8-12 Raw run-current register value. Useful when matching exact datasheet behavior or low-level tuning.
ihold_percent Library value Location: Settings Functions: setHoldCurrent(), setAllCurrentValues(), getSettings() 0% Derived from cached IHOLD_IRUN.IHOLD, bits 0-4 Approximate hold-current level expressed as a percentage. Controls standstill holding torque versus heat.
ihold_register_value Library value Location: Settings Functions: setHoldCurrent(), setAllCurrentValues(), getSettings() 0 Cached IHOLD_IRUN.IHOLD, bits 0-4 Raw hold-current register value. Useful for exact low-level tuning.
iholddelay_percent Library value Location: Settings Functions: setHoldDelay(), setAllCurrentValues(), getSettings() 6% Derived from cached IHOLD_IRUN.IHOLDDELAY, bits 16-19 Approximate delay or transition amount before current ramps down from run current to hold current. Helps avoid abrupt current drop right after motion stops.
iholddelay_register_value Library value Location: Settings Functions: setHoldDelay(), setAllCurrentValues(), getSettings() 1 Cached IHOLD_IRUN.IHOLDDELAY, bits 16-19 Raw hold-delay register value. Useful for precise tuning.
shaft Chip flag Location: GCONF / GlobalConfig Functions: enableInverseMotorDirection(), disableInverseMotorDirection() False GCONF.shaft, bit 3 Inverts logical motor direction. Possible values: False = normal direction, True = inverted direction. Lets you reverse direction in software instead of rewiring motor phases.
inverse_motor_direction_enabled Library value Location: Settings Functions: enableInverseMotorDirection(), disableInverseMotorDirection(), getSettings() False GCONF.shaft, bit 3 Library view of the direction inversion flag. Quick readback of the current direction polarity setting.
standstill_mode Library value Location: Settings Functions: setStandstillMode(), getSettings() NORMAL PWMCONF.freewheel, bits 20-21 How the driver behaves when standing still. Possible values: NORMAL, FREEWHEELING, STRONG_BRAKING, BRAKING. Changes whether the motor holds, coasts, or brakes at standstill.
i_scale_analog Chip flag Location: GCONF / GlobalConfig Functions: enableAnalogCurrentScaling(), disableAnalogCurrentScaling() False GCONF.i_scale_analog, bit 0 Uses analog VREF-based current scaling when enabled. Possible values: False = digital current scaling, True = analog VREF scaling. Important if your hardware relies on analog reference voltage for current control.
analog_current_scaling_enabled Library value Location: Settings Functions: enableAnalogCurrentScaling(), disableAnalogCurrentScaling(), getSettings() False GCONF.i_scale_analog, bit 0 Library view of whether analog current scaling is enabled. Helps confirm whether current is being set mainly by analog or digital means.
internal_rsense Chip flag Location: GCONF / GlobalConfig Functions: useInternalSenseResistors(), useExternalSenseResistors() False GCONF.internal_rsense, bit 1 Uses the internal current-sense path instead of external sense resistors. Possible values: False = external sense resistor path, True = internal sense path. Depends on board design and influences current measurement behavior.
internal_sense_resistors_enabled Library value Location: Settings Functions: useInternalSenseResistors(), useExternalSenseResistors(), getSettings() False GCONF.internal_rsense, bit 1 Library view of the internal sense-resistor setting. Useful confirmation when working with unusual hardware layouts.
automatic_current_scaling_enabled Library value Location: Settings Functions: enableAutomaticCurrentScaling(), disableAutomaticCurrentScaling(), getSettings() False PWMCONF.pwm_autoscale, bit 18 Reflects PWMCONF.pwm_autoscale. Possible values: False = manual amplitude behavior, True = automatic amplitude scaling. Usually helpful for quiet, stable motion without manual tuning.
automatic_gradient_adaptation_enabled Library value Location: Settings Functions: enableAutomaticGradientAdaptation(), disableAutomaticGradientAdaptation(), getSettings() False PWMCONF.pwm_autograd, bit 19 Reflects PWMCONF.pwm_autograd. Possible values: False = fixed gradient behavior, True = automatic gradient adaptation. Improves self-tuning behavior in StealthChop mode.
pwm_offset Library value Location: Settings Functions: setPwmOffset(), getSettings() 36 / 0x24 PWMCONF.pwm_offset, bits 0-7 Base PWM amplitude term used by StealthChop. Part of fine-grained acoustic and motion tuning.
pwm_gradient Library value Location: Settings Functions: setPwmGradient(), getSettings() 0 / 0x00 PWMCONF.pwm_grad, bits 8-15 Slope term for PWM adaptation in StealthChop. Another advanced tuning parameter for quiet motion quality.
cool_step_enabled Library value Location: Settings Functions: enableCoolStep(), disableCoolStep(), getSettings() False Library state; configured through COOLCONF Whether CoolStep adaptive current control is enabled. Possible values: False = CoolStep off, True = CoolStep enabled. CoolStep reduces current when load is low, cutting heat and power usage.
multistep_filt Chip flag Location: GCONF / GlobalConfig Functions: setup() True GCONF.multistep_filt, bit 8 Enables a filter on step pulses. Possible values: False = filter off, True = filter on. Can help reject glitches or noise on the step input.
index_otpw Chip flag Location: GCONF / GlobalConfig Functions: no dedicated public setter in this library False GCONF.index_otpw, bit 4 Changes the function of the INDEX output related to overtemperature prewarning behavior. Possible values: False = normal INDEX behavior, True = OTPW-related behavior. Mostly relevant only if you actively use the INDEX output pin.
index_step Chip flag Location: GCONF / GlobalConfig Functions: no dedicated public setter in this library False GCONF.index_step, bit 5 Changes the function of the INDEX output to a step-related indication. Possible values: False = normal INDEX behavior, True = STEP-related behavior. Again only relevant when INDEX output is used in hardware.
test_mode Chip flag Location: GCONF / GlobalConfig Functions: no public application API; leave unused False GCONF.test_mode, bit 9 Internal test mode bit. Possible values: False = normal operation, True = factory test behavior. Do not use this in normal applications.
reserved Chip bits Location: remaining GCONF bits Functions: none; leave untouched Keep default Reserved bits Reserved bits in the register. Leave untouched unless the datasheet explicitly says otherwise.

Note: the defaults shown here are the library's effective post-setup() values, not the raw silicon reset values. Current-related library values in this library are based on cached settings, not a fresh IHOLD_IRUN readback. A few effective defaults differ from named constants because initialize() first writes packed register defaults and then applies extra changes such as minimizing current, disabling software output, and disabling PWM autoscale/autograd.

Plain-English Glossary

  • Chopper: The internal current-control engine. It rapidly switches the motor power transistors on and off so the coil current follows the requested target.
  • SpreadCycle: Trinamic’s more torque-oriented and dynamic current-control mode. Usually stronger at speed/load, but not as quiet as StealthChop.
  • StealthChop: Trinamic’s very quiet PWM-based mode. Great for low noise, usually less aggressive than SpreadCycle under demanding motion.
  • CoolStep: Automatic load-based current adjustment. The driver reduces current when the motor does not need full torque and raises it again when load increases.
  • Current scaling: The mechanism that determines how much motor current the driver delivers. This depends on IRUN, IHOLD, VREF behavior, sense resistors, and related options.
  • Autoscale: In StealthChop, the chip automatically adjusts PWM/current regulation amplitude so you do not have to tune everything manually.
  • Autograd: Automatic tuning of the PWM gradient term used by StealthChop.
  • Sense resistor / Rsense: The resistor path used to measure motor current. Some designs use external resistors; the TMC2209 also supports an internal sense option.

Practical Overview

For most real projects, the most important items are is_communicating, is_setup, pdn_disable, mstep_reg_select, microsteps_per_step, enable_spread_cycle, stealth_chop_enabled, and the current-related fields irun/ihold.

If UART communication is working but microstep changes do nothing, the first thing to check is mstep_reg_select. If it is False, the hardware pins MS1/MS2 still decide the microstep resolution. If it is True, the register CHOPCONF.MRES decides instead.

For silent operation, StealthChop-related settings are the main focus. For stronger high-load behavior, SpreadCycle and current-related tuning become more important. CoolStep is useful when you want to reduce heat and power consumption automatically when the motor is lightly loaded.
