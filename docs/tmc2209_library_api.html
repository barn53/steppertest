TMC2209 Library API Reference

This page describes the public, user-facing API of the local janelia-arduino/TMC2209 library used in this project. Each row explains what the function does, which chip registers it reads or writes, what defaults or side effects are involved, and what dependencies or caveats matter in real use. Technical phrases like current scaling, StealthChop, SpreadCycle, and CoolStep are translated into simpler language in the explanations and glossary below.

Setup And Basic Control

Function Function Parameters Chip Registers Touched / Read Explanation Dependencies / Considerations
setup(...) Overloads for HardwareSerial, platform-specific UART types, and SoftwareSerial serial
serial_baud_rate = 115200 for hardware UARTs
serial_baud_rate = 9600 for SoftwareSerial
serial_address = SERIAL_ADDRESS_0
ESP32 / RP2040 overloads also allow alternate_rx_pin, alternate_tx_pin 		Writes: GCONF, IHOLD_IRUN, CHOPCONF, PWMCONF, COOLCONF, GSTAT, TPOWERDOWN, TPWMTHRS, VACTUAL, TCOOLTHRS, SGTHRS
Effective post-setup defaults include: GCONF.pdn_disable=True, GCONF.mstep_reg_select=True, GCONF.multistep_filt=True, CHOPCONF.MRES=256, IHOLD_IRUN.IRUN=0, IHOLD_IRUN.IHOLD=0, IHOLD_IRUN.IHOLDDELAY=1, CHOPCONF.toff=0, PWMCONF.pwm_autoscale=False, PWMCONF.pwm_autograd=False 		Starts the serial connection to the driver and runs the library's initialization sequence. it switches the chip into UART-controlled mode, writes the library's preferred starting configuration, clears any old drive-error latch, reduces motor current to the minimum, and leaves the driver software-disabled. This is the most important call; most other functions assume it has already succeeded. On ESP32, the local patched copy intentionally avoids calling Serial.end() before begin() in the generic path because that caused the first GCONF write to fail on ESP32-C3. For UART control to work fully, the wiring, board routing, and UART address must also be correct.
setHardwareEnablePin() hardware_enable_pin No chip register write. MCU GPIO only. Tells the library which microcontroller pin is connected to the driver's hardware EN input. The function immediately configures that MCU pin as an output and drives it HIGH, which means the driver starts out hardware-disabled. This does not replace enable() or disable(); it adds hardware-level control on top of the chip's software enable state. If you use this, remember that a HIGH level on EN still disables the power stage.
enable() No parameters Writes CHOPCONF.toff via CHOPCONF; may also drive the MCU hardware enable pin LOW Turns the power stage on in normal library use. The driver can actively energize the motor coils again. If a hardware enable pin was configured, this function also pulls it LOW. It does not change motor current settings; it only restores the saved chopper off-time. If hardwareDisabled() stays True, the chip is still blocked externally.
disable() No parameters Writes CHOPCONF.toff = 0 via CHOPCONF; may also drive the MCU hardware enable pin HIGH Turns the power stage off in software. The driver stops actively driving the motor coils. This is the library's default state after setup(). If you need the motor to hold or move, you usually need enable() afterward.

Motion And Current Configuration

Function Function Parameters Chip Registers Touched / Read Explanation Dependencies / Considerations
setMicrostepsPerStep() microsteps_per_step
Valid values are powers of two from 1 to 256; other values are rounded down		 Writes CHOPCONF.MRES via CHOPCONF Sets how many tiny electrical sub-steps the driver divides one full motor step into. Higher microstep values make motion smoother and quieter, but they also require a higher step rate from the controller. This only controls the chip if GCONF.mstep_reg_select=True, which setup() normally sets. If that bit is False, the physical MS1/MS2 pins still decide the microstep value instead.
setMicrostepsPerStepPowerOfTwo() exponent
0..8, where 0=1, 1=2, ..., 8=256		 Writes CHOPCONF.MRES via CHOPCONF Same goal as setMicrostepsPerStep(), but it uses the power-of-two exponent directly instead of the human-readable microstep count. Useful when you already think in 2^n form. The same mstep_reg_select dependency applies here too.
setRunCurrent() percent
0..100		 Writes IHOLD_IRUN.IRUN via IHOLD_IRUN; may also update COOLCONF.seimin if CoolStep is enabled Sets the motor's running current. This changes how hard the driver pushes current into the motor while it is moving, which strongly affects available torque, heat, and power draw. The library maps percent to the chip's 0..31 current scale. The current-related values returned by getSettings() come from the library's cached copy, not a fresh read of IHOLD_IRUN.
setHoldCurrent() percent
0..100		 Writes IHOLD_IRUN.IHOLD via IHOLD_IRUN Sets the standstill current. This controls how strongly the motor holds position while not moving. Higher hold current gives stronger holding torque but creates more heat at standstill.
setHoldDelay() percent
0..100		 Writes IHOLD_IRUN.IHOLDDELAY via IHOLD_IRUN Sets how gradually the chip drops from run current to hold current after motion stops. It softens the transition so the driver does not immediately jump to the lower hold current. This does not directly change torque by itself; it changes the delay and ramp-down behavior between moving and standing still.
setAllCurrentValues() run_current_percent
hold_current_percent
hold_delay_percent
All 0..100 		Writes IHOLD_IRUN once Convenience call that sets run current, hold current, and hold delay in one write. This is the cleanest way to update all three current-related settings together when you want them to stay consistent.
setRMSCurrent() mA, rSense, holdMultiplier = 0.5 Writes CHOPCONF.vsense via CHOPCONF, then writes IHOLD_IRUN.IRUN and IHOLD_IRUN.IHOLD Sets current from a target RMS current in milliamps instead of a simple percent. It also automatically chooses the high-sensitivity current measurement mode when needed. rSense must match the board's real sense resistor value if you want the current calculation to be meaningful. This function may call the chip's vsense mode on or off automatically.
enableDoubleEdge() / disableDoubleEdge() No parameters Writes CHOPCONF.dedge via CHOPCONF Controls whether the STEP interface reacts to both edges of the step pulse instead of just one. Use this only if your motion-controller timing actually expects double-edge stepping. Otherwise, leave it off.
enableVSense() / disableVSense() No parameters Writes CHOPCONF.vsense via CHOPCONF Switches the chip between normal and high-sensitivity current measurement range. it changes how the driver measures coil current internally. Most users do not need to toggle this directly because setRMSCurrent() already chooses it automatically when useful.
enableInverseMotorDirection() / disableInverseMotorDirection() No parameters Writes GCONF.shaft via GCONF Reverses the logical motor direction in software. It flips "forward" and "backward" without rewiring the motor. This changes the driver's interpretation of direction. It does not change any wiring or motor current settings.
setStandstillMode() mode: NORMAL, FREEWHEELING, STRONG_BRAKING, BRAKING Writes PWMCONF.freewheel via PWMCONF Chooses what the driver should do electrically when the motor is standing still: hold normally, coast freely, or actively brake. This strongly affects how the motor behaves at zero speed. FREEWHEELING usually removes holding force, while braking modes can stop the shaft from coasting.
enableAutomaticCurrentScaling() / disableAutomaticCurrentScaling() No parameters Writes PWMCONF.pwm_autoscale via PWMCONF Controls StealthChop's automatic amplitude adjustment. It decides whether the chip should tune its own PWM drive strength automatically instead of relying only on fixed manual values. The local library's setup() leaves this disabled. This matters mainly when you are using StealthChop.
enableAutomaticGradientAdaptation() / disableAutomaticGradientAdaptation() No parameters Writes PWMCONF.pwm_autograd via PWMCONF Controls StealthChop's automatic gradient tuning. It lets the chip auto-tune how aggressively the PWM drive changes with speed and load. Like autoscale, this is mainly relevant in StealthChop mode. The local library's setup() leaves it disabled.
setPwmOffset() pwm_amplitude
0..255		 Writes PWMCONF.pwm_offset via PWMCONF Sets the base PWM amplitude used by StealthChop. This is part of the low-level quiet-motion tuning. Most users can leave this alone unless they are deliberately tuning StealthChop behavior.
setPwmGradient() pwm_amplitude
0..255		 Writes PWMCONF.pwm_grad via PWMCONF Sets the PWM gradient term used by StealthChop. It changes how quickly the PWM behavior ramps with operating conditions. Another advanced tuning knob. The local library's setup() leaves the effective value at 0.
setPowerDownDelay() power_down_delay
Library default written during setup(): 20		 Writes TPOWERDOWN Sets how long the chip waits before entering its power-down behavior after standstill. This is mostly useful when balancing quick wake-up against power-saving behavior at idle.
setReplyDelay() delay
Clamped to 0..15 (REPLY_DELAY_MAX)		 Writes REPLYDELAY.replydelay via REPLYDELAY Sets the chip's UART reply delay. This adds a short wait before the driver answers a read request. This can help in multi-driver or timing-sensitive UART setups. The header notes that values of at least 2 are useful when using multiple serial addresses with bidirectional communication.

Motion Modes, Thresholds, And Load Features

Function Function Parameters Chip Registers Touched / Read Explanation Dependencies / Considerations
moveAtVelocity() microsteps_per_period Writes VACTUAL Commands the chip's internal velocity mode. You tell the driver "spin this fast" instead of supplying external STEP pulses. This is different from classic STEP/DIR control. The sign sets direction; the magnitude sets speed.
moveUsingStepDirInterface() No parameters Writes VACTUAL = 0 Returns control to the usual external STEP/DIR interface. Use this after moveAtVelocity() if you want the external step pin to drive motion again.
enableStealthChop() / disableStealthChop() No parameters Writes GCONF.enable_spread_cycle via GCONF Selects the driver's main current-control style. enableStealthChop() chooses the quieter PWM-based mode. disableStealthChop() switches to SpreadCycle, which is usually the more forceful and dynamic mode. StealthChop is usually chosen for quiet motion; SpreadCycle is often preferred when torque, high-speed behavior, or dynamic response matter more than silence.
setStealthChopDurationThreshold() duration_threshold Writes TPWMTHRS Sets the threshold that influences when StealthChop should be active. This only matters if you are using StealthChop. The local library's setup() leaves the threshold at 0.
setStallGuardThreshold() stall_guard_threshold Writes SGTHRS Sets the stall-detection sensitivity. This adjusts how easily the driver decides the motor is heavily loaded or stalling. This setting becomes useful only if you actively use StallGuard-related feedback such as getStallGuardResult() or CoolStep.
enableCoolStep() / disableCoolStep() Enable: lower_threshold = 1, upper_threshold = 0
Disable: no parameters 		Writes COOLCONF.semin and COOLCONF.semax via COOLCONF; also updates library state cool_step_enabled_ Turns CoolStep on or off. CoolStep means the chip automatically lowers current when the motor load is light and raises it when load increases. It is an automatic "use only as much current as needed" feature. The library stores a separate boolean for this feature and reports it in getSettings(). In practical use, CoolStep is usually configured together with StallGuard-related thresholds such as SGTHRS and TCOOLTHRS.
setCoolStepCurrentIncrement() current_increment: CURRENT_INCREMENT_1, _2, _4, _8 Writes COOLCONF.seup via COOLCONF Sets how quickly CoolStep is allowed to raise current when more torque is needed. This matters only if CoolStep is in use. Larger increments react faster but can be less gentle.
setCoolStepMeasurementCount() measurement_count: MEASUREMENT_COUNT_32, _8, _2, _1 Writes COOLCONF.sedn via COOLCONF Sets how many load measurements CoolStep considers before reducing current. Smaller measurement counts react more quickly; larger ones react more conservatively.
setCoolStepDurationThreshold() duration_threshold Writes TCOOLTHRS Sets the duration threshold associated with CoolStep and load-based regulation. This is usually part of the same tuning bundle as CoolStep and StallGuard, not an isolated one-off setting.
enableAnalogCurrentScaling() / disableAnalogCurrentScaling() No parameters Writes GCONF.i_scale_analog via GCONF Chooses whether motor current is mainly scaled from the analog VREF input or from digital register values. This decides whether a voltage on the board controls current scaling or the UART register settings do. The local library's setup() leaves this disabled, which means digital register control is used.
useExternalSenseResistors() / useInternalSenseResistors() No parameters Writes GCONF.internal_rsense via GCONF Chooses whether the chip should use the board's external current-sense resistors or its internal sensing path. This changes how the driver measures motor current. This depends heavily on the actual board design. Most StepStick-style modules are built around external sense resistors, so do not switch this casually without understanding the hardware.

Status, Readback, And Diagnostics

Function Function Parameters Chip Registers Touched / Read Explanation Dependencies / Considerations
getVersion() No parameters Reads IOIN.version via IOIN Returns the chip's version byte. The library expects 0x21 for a TMC2209. This is the low-level read used by isCommunicating().
isCommunicating() No parameters Reads IOIN.version via getVersion() Checks whether UART communication is working at all. If this returns False, do not trust other UART-based results yet. Power, address, wiring, and serial configuration are the first things to check.
isSetupAndCommunicating() No parameters Reads GCONF.pdn_disable via GCONF Library health check that mainly answers: "Is the chip in the expected UART setup mode?" Important nuance: despite the name, this function does not separately verify the version byte first; it mainly checks whether pdn_disable reads back as True.
isCommunicatingButNotSetup() No parameters Indirectly uses IOIN.version and GCONF.pdn_disable Reports the mixed state where UART communication works, but the chip is not in the library's expected setup mode. This can happen if the driver reset after setup or if setup bits like pdn_disable did not stick.
hardwareDisabled() No parameters Reads IOIN.enn via IOIN Checks the hardware enable input state. It answers whether the chip is being held disabled by the physical EN pin. This is separate from the software-enabled state controlled by enable() / disable(). You can have working UART and still be hardware-disabled.
getMicrostepsPerStep() No parameters Uses cached CHOPCONF.MRES already stored in the library Returns the human-readable microstep value, such as 16 or 256. This method does not perform a fresh register read by itself. It interprets the library's cached chopper_config_, so getSettings() is the safer "freshen then summarize" API.
getSettings() No parameters Reads GCONF, CHOPCONF, PWMCONF
Also uses cached library values for IHOLD_IRUN and cool_step_enabled_ 		Returns a high-level summary struct with the settings most applications care about, such as communication state, software enable state, microsteps, direction inversion, current values, PWM tuning, CoolStep state, and sense mode. The current-related fields are not freshly read from IHOLD_IRUN; they come from the library's cached copy. That is important if some other code or another MCU changes the chip behind the library's back.
getStatus() No parameters Reads DRV_STATUS Returns the detailed driver-status struct, including overtemperature flags, short/open-load flags, current scaling, StealthChop mode, and standstill state. Good for fault diagnosis and deeper tuning. The current_scaling field means the chip's instantaneous internal current-scale value, not just your commanded IRUN setting.
getGlobalStatus() No parameters Reads GSTAT Returns the global status latch with flags like reset, drv_err, and uv_cp. These bits are useful when debugging resets, driver faults, or undervoltage-related events.
clearReset() No parameters Writes GSTAT.reset = 1 via GSTAT Clears the latched reset flag. The reset bit is sticky, so once it is set it can keep reading True until you clear it explicitly.
clearDriveError() No parameters Writes GSTAT.drv_err = 1 via GSTAT Clears the latched drive-error flag. The local setup() already does this once during initialization.
getInterfaceTransmissionCounter() No parameters Reads IFCNT Returns the chip's UART write-counter value. This is excellent for debugging because it increases when valid UART write frames are accepted.
getInterstepDuration() No parameters Reads TSTEP Returns the measured time between incoming STEP pulses. This is useful only when the STEP/DIR interface is actually being driven.
getStallGuardResult() No parameters Reads SG_RESULT Returns the chip's load-measurement result used by StallGuard. Higher-level meaning depends on your motor, speed, and StallGuard configuration; it is a raw measurement, not a simple yes/no stall answer.
getPwmScaleSum() No parameters Reads PWM_SCALE.pwm_scale_sum via PWM_SCALE Returns one of the chip's internal PWM scaling readbacks used in StealthChop. Mostly useful for advanced tuning or diagnostics, not for ordinary motor control logic.
getPwmScaleAuto() No parameters Reads PWM_SCALE.pwm_scale_auto via PWM_SCALE Returns the automatic PWM scale value currently chosen by the chip. This is mainly meaningful when automatic StealthChop features are in use.
getPwmOffsetAuto() No parameters Reads PWM_AUTO.pwm_offset_auto via PWM_AUTO Returns the automatically determined PWM offset value. Advanced diagnostic readback; most users can ignore it unless tuning StealthChop deeply.
getPwmGradientAuto() No parameters Reads PWM_AUTO.pwm_gradient_auto via PWM_AUTO Returns the automatically determined PWM gradient value. Advanced diagnostic readback paired with getPwmOffsetAuto().
getMicrostepCounter() No parameters Reads MSCNT Returns the chip's internal microstep position counter. Useful for low-level diagnostics or synchronization, especially when you want to know where the driver currently is within the microstep cycle.

Note: functions like getSettings() and getMicrostepsPerStep() mix direct register reads with cached library state. That is accurate for how this library works, but it is different from a "read every register fresh every time" design.

Plain-English Glossary

  • Current scaling: The chip's way of deciding how much motor current to push into the coils. This is the "how strong / how hot / how much torque" setting.
  • StealthChop: Trinamic's very quiet drive mode. It is usually chosen when low noise matters more than maximum dynamic response.
  • SpreadCycle: Trinamic's more forceful current-control mode. It is often better when speed changes, load response, or torque matter more than silence.
  • Chopper: The internal current-control engine that rapidly switches the output transistors on and off so the winding current stays near the target value.
  • PWM offset: A base drive level used by StealthChop. Think of it as one part of the quiet-motion tuning baseline.
  • PWM gradient: A slope value that changes how the PWM behavior ramps with speed and load.
  • Autoscale: The chip automatically adjusts PWM drive strength for you instead of relying only on fixed manual values.
  • Autograd: The chip automatically adjusts the PWM gradient for you.
  • CoolStep: Automatic load-based current saving. The driver uses less current when full force is not needed and raises it again when load increases.
  • StallGuard: A load-measurement feature used to estimate how hard the motor is working and whether it may be nearing a stall.
  • Sense resistor / Rsense: The resistor path used to measure motor current. Some boards use external resistors; the chip can also use an internal sense path in some hardware designs.
  • VSense: A setting that changes the current-measurement sensitivity range used by the driver.
